บันทึกเวลารอเอกซเรย์ — เวอร์ชันเชื่อม Google Sheets
เว็บ static (เปิดบน GitHub Pages) + Google Apps Script เป็น API + Google Sheets เป็นฐานข้อมูล
มีระบบ login ด้วยรหัสผ่านรวมของแผนก และทำงานแบบ เก็บในเครื่องก่อน แล้วซิงก์ขึ้น Sheet (ใช้ต่อได้แม้เน็ตหลุด)
ไฟล์ในชุดนี้
`index.html` — หน้าเว็บ (อัปขึ้น GitHub)
`Code.gs` — โค้ดฝั่ง Google Apps Script (วางใน Sheet)
`README.md` — คู่มือนี้
---
ส่วนที่ 1 — ตั้งค่าฝั่ง Google (ทำครั้งเดียว)
สร้าง Google Sheet ใหม่ 1 ไฟล์ (จะเป็นฐานข้อมูล) ตั้งชื่ออะไรก็ได้ เช่น "ฐานข้อมูลเวลารอเอกซเรย์"
ในไฟล์นั้น เมนู Extensions › Apps Script
ลบโค้ดเดิมในตัวแก้ไข แล้ว วางเนื้อหาไฟล์ `Code.gs` ทั้งหมด ลงไป → กด Save (💾)
แก้รหัสผ่าน: ในฟังก์ชัน `setup()` เปลี่ยนค่า `'CHANGE_ME_1234'` เป็นรหัสผ่านของแผนกที่ต้องการ
เลือกฟังก์ชัน `setup` จากแถบด้านบน แล้วกด Run หนึ่งครั้ง
ครั้งแรกจะขึ้นให้ อนุญาตสิทธิ์ (Authorize) → เลือกบัญชี → Advanced › Go to (project) › Allow
เมื่อรันสำเร็จ จะสร้างชีตชื่อ `records` และตั้งรหัสผ่านให้อัตโนมัติ
Deploy › New deployment
ไอคอนเฟือง (⚙) เลือก Web app
Execute as: Me
Who has access: Anyone
กด Deploy → คัดลอก Web app URL (ลงท้ายด้วย `/exec`) เก็บไว้
> ตรวจว่าใช้ได้: เปิด URL นั้นในเบราว์เซอร์ ควรเห็นข้อความ `{"ok":true,"service":"xray-wait",...}`

ส่วนที่ 2 — เอาเว็บขึ้น GitHub Pages
ไฟล์เว็บชื่อ `index.html` อยู่แล้ว (เปิดที่ URL หลักได้เลย)
สร้าง repository ใหม่บน GitHub → อัป `index.html` (ลาก-วางในหน้า repo ได้)
Settings › Pages › Source: `main` / root › Save
รอสักครู่ จะได้ลิงก์ `https://<username>.github.io/<repo>/`

ส่วนที่ 3 — เชื่อมเว็บกับเซิร์ฟเวอร์ (ทำครั้งแรกในแต่ละเครื่อง/เบราว์เซอร์)
เปิดลิงก์เว็บ → หน้า login
กด ⚙ ตั้งค่าที่อยู่เซิร์ฟเวอร์ → วาง Web app URL (`/exec`) → บันทึก
กรอก รหัสผ่านของแผนก → เข้าสู่ระบบ
เครื่องนั้นจะจำที่อยู่เซิร์ฟเวอร์ไว้ ครั้งต่อไปแค่กรอกรหัสผ่าน (login ค้างไว้ ~12 ชั่วโมง)
---
การใช้งาน
หน้ากรอกข้อมูล: เลือกวันที่ (ย้อนหลังได้) → กรอก HN, เวลาเริ่มรอ, เวลาเสร็จ → กดบันทึก → กด"ถัดไป" เพื่อขึ้นวันใหม่
การซิงก์: ทุกครั้งที่บันทึก/แก้/ลบ ข้อมูลจะลงเครื่องทันที และระบบพยายามซิงก์ขึ้น Sheet ให้เอง
ปุ่ม ซิงก์ มุมขวาบนบอกสถานะ: `ซิงก์แล้ว` / `ซิงก์ (N รอ)` = มี N รายการยังไม่ขึ้น Sheet
เน็ตหลุด → ข้อมูลอยู่ครบในเครื่อง พอมีเน็ตกดปุ่มซิงก์เพื่อส่งขึ้น Sheet
หน้าสรุปผล: KPI, กราฟเวลารอเฉลี่ยรายวัน (เส้นเป้า 15 นาที), การ์ดรายวัน, ตารางรวม (แก้/ลบได้), ปุ่มส่งออก Excel
> หลายคนใช้พร้อมกันได้ แต่ต้อง **กดซิงก์** เพื่อให้เห็นข้อมูลของกันและกัน (ระบบดึงข้อมูลล่าสุดจาก Sheet ตอนซิงก์)

ความปลอดภัยและข้อควรระวัง
HN เป็นข้อมูลระบุตัวผู้ป่วย เมื่อเก็บใน Google Sheets = ข้อมูลอยู่บนคลาวด์
ตั้งสิทธิ์การแชร์ Sheet ให้ เฉพาะคนในแผนกที่จำเป็น (อย่าตั้ง "ทุกคนที่มีลิงก์")
พิจารณา PDPA ของโรงพยาบาล เก็บเท่าที่จำเป็น และมีผู้รับผิดชอบดูแลไฟล์
login แบบรหัสผ่านรวมเป็นการกันคนนอกระดับใช้งานภายใน ไม่ใช่ระดับธนาคาร
เปลี่ยนรหัสผ่านเป็นระยะ (รัน `setPassword("รหัสใหม่")` ใน Apps Script)
ไม่ควรใช้รหัสเดียวกับระบบสำคัญอื่น
Web app URL เป็นสาธารณะ แต่ทุก action (ยกเว้น login) ต้องมี token ที่ได้จากการใส่รหัสผ่านถูกก่อน จึงเข้าถึงข้อมูลไม่ได้ถ้าไม่รู้รหัส
---
แก้ไขโค้ด Apps Script ภายหลัง
ทุกครั้งที่แก้ `Code.gs` ต้อง deploy เวอร์ชันใหม่:
Deploy › Manage deployments › (ดินสอ ✏ แก้ไข) › Version: New version › Deploy
(URL เดิมใช้ได้ต่อ ไม่ต้องเปลี่ยนในเว็บ)
---
ปัญหาที่พบบ่อย
`bad_password` — รหัสผ่านไม่ตรงกับที่ตั้งใน `setup()`
`no_password_set` — ยังไม่ได้รัน `setup()` หรือยังไม่ได้ตั้ง `APP_PASSWORD`
เชื่อมต่อไม่ได้ / เซิร์ฟเวอร์ตอบกลับไม่ถูกต้อง — มักเกิดจากยังไม่ได้ Deploy เป็น Web app, ตั้ง "Who has access" ไม่ใช่ Anyone, หรือแก้โค้ดแล้วยังไม่ deploy เวอร์ชันใหม่
ข้อมูลของเพื่อนร่วมงานไม่ขึ้น — กดปุ่มซิงก์ (ระบบดึงข้อมูลล่าสุดจาก Sheet ตอนซิงก์)

หมายเหตุ
ข้อมูลเดิมที่เคยบันทึกในเวอร์ชันออฟไลน์ (เก็บในเครื่อง) จะถูกย้ายเข้าระบบให้อัตโนมัติเมื่อเปิดครั้งแรก และรอสถานะ "ซิงก์" เพื่อส่งขึ้น Sheet
อยากได้ PWA (เพิ่มลงหน้าจอโฮมมือถือ + ใช้ออฟไลน์เต็มรูปแบบ) บอกได้ เพิ่ม `manifest.json` + service worker ให้ได้
